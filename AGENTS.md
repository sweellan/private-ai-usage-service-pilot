# Vibe Usage Privacy Check Rules

## 对象定位
- 规则：本对象用于验证 `@vibe-cafe/vibe-usage` 的源码、命令行为与数据出口，判断它是否适合只做本地统计。
  - 原因：这次任务的关键不是“能不能跑起来”，而是“跑的同时会不会把数据发出去”。

## 当前工作方式
- 规则：所有外部源码、包元信息、命令探针与网络行为证据都优先沉淀在 `runs/`。
  - 原因：这类调查更像批次化验证，适合按 run 回看。
- 规则：在没有确认安全边界前，不对真实本地 usage 数据直接执行可能触发上传的正式命令。
  - 原因：隐私验证不能通过先上传一次再回头复盘来完成。

## 通用执行规则
- 规则：先做源码级检查与帮助信息检查，再决定是否做最小运行验证。
  - 原因：先确认命令和默认行为，可以减少不必要的数据外发风险。
- 规则：如果发现官方只支持云端同步，不直接改写上游源码；先给出本地替代路径与风险说明。
  - 原因：当前任务是可行性判断，不是立即维护一个私有分叉。
